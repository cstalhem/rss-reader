# Technology Stack

**Analysis Date:** 2026-02-04

## Languages

**Primary:**
- Python 3.14+ - Backend API and RSS feed processing
- TypeScript 5.x - Frontend UI and Next.js app
- JavaScript (ES2017+) - React/Next.js runtime

**Secondary:**
- CSS - Frontend styling

## Runtime

**Environment:**
- Node.js (latest) - Frontend development and production
- Python 3.14+ - Backend runtime

**Package Manager:**
- `npm` or `yarn` - JavaScript/TypeScript dependency management (frontend)
- `uv` (UV build backend) - Python dependency management and virtual environment (backend)
- Lockfile: `uv.lock` (backend), `package-lock.json` (frontend, generated by npm)

## Frameworks

**Core:**
- FastAPI 0.128.0+ - Python REST API framework and request handling
- Next.js 16.1.6 - React meta-framework for frontend with App Router
- React 19.2.3 - UI library for frontend
- React DOM 19.2.3 - React DOM renderer for browser

**Data:**
- SQLModel 0.0.32+ - Python ORM combining SQLAlchemy and Pydantic for database models and validation
- SQLAlchemy (via SQLModel) - SQL toolkit and ORM

**Background Jobs:**
- APScheduler 3.11.2+ - Python task scheduler for periodic feed refresh jobs

**HTTP/Network:**
- httpx 0.28.1+ - Python async HTTP client for fetching RSS feeds
- Uvicorn 0.40.0+ - Python ASGI web server for FastAPI

## Key Dependencies

**Critical:**
- feedparser 6.0.12+ - RSS/Atom feed parsing library for Python
- FastAPI 0.128.0+ - Core API framework handling all HTTP requests and endpoints
- SQLModel 0.0.32+ - Unified data modeling and database interaction
- APScheduler 3.11.2+ - Enables background feed refresh job execution every 30 minutes

**Frontend:**
- next 16.1.6 - Framework providing routing, rendering, and build tooling
- react 19.2.3 - UI library with hooks and functional components
- react-dom 19.2.3 - React rendering engine

**Development:**
- TypeScript 5.x - Static type checking for frontend code
- ESLint 9.x - Linting for frontend code
- eslint-config-next 16.1.6 - Next.js ESLint configuration and rules
- pytest 9.0.2+ - Python testing framework
- pytest-asyncio 1.3.0+ - Pytest plugin for async test support
- ruff 0.15.0+ - Python linter and formatter (enforces PEP 8)

## Configuration

**Environment:**
- `.python-version` - Specifies Python 3.14 as required backend version
- No `.env` files required currently; hardcoded defaults used for development
- CORS configured to allow `http://localhost:3210` frontend during development
- Feed refresh interval hardcoded to 30 minutes via APScheduler

**Build:**
- `backend/pyproject.toml` - Python project configuration, dependencies, and tool settings (ruff, pytest)
- `frontend/tsconfig.json` - TypeScript compiler options with path aliases (`@/*`)
- `frontend/package.json` - JavaScript dependencies and npm scripts
- `frontend/next.config.ts` - Next.js configuration (currently minimal)

**Ruff Configuration (Backend):**
- Line length: 88 characters
- Target version: Python 3.14
- Lint rules: E (pycodestyle errors), W (warnings), F (Pyflakes), I (isort), B (flake8-bugbear), C4 (comprehensions), UP (pyupgrade)
- Ignored: E501 (long lines), B008 (FastAPI Depends pattern)
- Quote style: Double quotes
- Indent: Spaces

## Platform Requirements

**Development:**
- Python 3.14+ installed and available in PATH
- Node.js with npm or yarn installed
- `uv` package manager for Python (handles virtual environments and dependencies)
- Unix-like shell (Bash/Zsh) for running commands

**Production:**
- Python 3.14+ runtime
- Node.js runtime for Next.js server (if not using static export)
- SQLite 3.x (embedded in Python, no external installation needed)
- File system access for `./data/` directory to store SQLite database

## Database

**Type:**
- SQLite 3.x (local file-based SQL database)

**Storage:**
- Location: `./data/rss.db` (relative to backend working directory)
- Schema: Managed by SQLModel ORM (auto-creates tables on startup)
- Tables: `feeds` (RSS feed sources), `articles` (parsed feed articles)

## API Specification

**Backend API:**
- Running on port 8912 during development (via `uvicorn backend.main:app --reload --port 8912`)
- FastAPI auto-generates OpenAPI docs at `/docs` (Swagger UI)
- Base URL: `http://localhost:8912`
- Endpoints: See `/api/articles`, `/api/feeds/refresh`, etc.

**Frontend Port:**
- Running on port 3210 during development (via `next dev`)
- CORS allows requests from frontend to `http://localhost:8912`
